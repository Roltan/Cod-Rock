# Тестовое задание: Интернет-магазин с логистикой

## Описание проекта

Это учебный проект интернет-магазина с системой доставки товаров. Основная фишка — расчёт маршрутов доставки от склада производителя до пункта выдачи заказов (ПВЗ) с учётом разных видов транспорта и критериев оптимизации.

**Что есть:**
- ✅ Готовый фронтенд (билд)
- ✅ Swagger документация API (`swagger.yaml`)
- ✅ Референсная реализация на Flask (этот репозиторий)
- ✅ Граф дорог России из 520 маршрутов (`instance/map.json`)

**Твоя задача:**
Написать бэкенд **с нуля**, который реализует API из Swagger документации.

---

## Архитектура системы

### Роли пользователей

**1. Покупатель (user)**
- Регистрация/авторизация
- Просмотр каталога товаров
- Добавление в корзину
- Расчёт маршрутов доставки
- Оформление заказа

**2. Производитель (producer)**
- Регистрация/авторизация
- Управление своими товарами (CRUD)
- Управление складами
- Управление пунктами выдачи заказов (ПВЗ)

### База данных

Структура таблиц (`instance/models.py`):

**User** — покупатели
- id, name, password (hash)

**Producer** — производители
- id, name, password (hash)

**Stuff** — товары
- id, name, photo, price, size, mass, description, producer (FK)

**Storehouse** — склады производителей
- id, city, producer (FK)

**PVZ** — пункты выдачи заказов
- id, producer (FK), city, address
- time_from, price_from, distance_from — доп. затраты на доставку до ПВЗ

**Orders** — заказы
- id, user (FK), stuff (FK), status
- storehouse (FK), pvz (FK)
- initial_city, final_city, way — данные о маршруте

**Map** — граф дорог между городами
- id, initial_city, final_city
- time_way, price_way, distance_way
- transport (car/air/ship)

## Авторизация

Используется **JWT токены** (Flask-JWT-Extended в референсе).

### Регистрация `PUT /register`
```json
{
  "name": "username",
  "password": "password",
  "role": "user" // или "producer"
}
```
- Проверка что пользователь с таким именем не существует (ни в User, ни в Producer)
- Хеширование пароля (Werkzeug в референсе: `generate_password_hash`)
- Создание записи в соответствующей таблице
- Возврат JWT токена

### Вход `POST /login`
```json
{
  "name": "username",
  "password": "password"
}
```
- Поиск пользователя сначала в User, потом в Producer
- Проверка пароля (`check_password_hash`)
- Возврат JWT токена

### Защищённые эндпоинты
Требуют заголовок: `Authorization: Bearer <token>`

JWT токен содержит `identity=name` (имя пользователя).

**Важно:** Некоторые эндпоинты доступны только определённым ролям:
- `/cart`, `/getWay`, `/acceptOrder` — только для user
- `/galaryProducer`, `/pvz/Producer`, `/Storehouse/Producer` — только для producer

## Граф дорог и инициализация данных

### Файл `instance/map.json`

Содержит **520 маршрутов** между городами России:
```json
{
  "id": 1,
  "initial_city": "Москва",
  "final_city": "Санкт-Петербург",
  "time_way": 630,      // минуты
  "price_way": 4000,    // рубли
  "distance_way": 650   // километры
}
```

### Загрузка в БД

При первом запуске нужно загрузить дороги в таблицу Map:

```python
# В main.py раскомментировать строку 39
СreateMap()
```

Функция читает `map.json` и для каждой дороги случайно выбирает транспорт:
- 70% — машина (car)
- 10% — самолёт (air)
- 20% — корабль (ship)

**После первого запуска закомментировать обратно**, иначе при каждом старте будут дубликаты.

Или добавь проверку:
```python
if Map.query.count() == 0:
    СreateMap()
```

## Ключевая задача: Поиск маршрута доставки

### Постановка задачи

**Дано:**
- Граф дорог между городами (таблица Map)
- Начальная точка: ПВЗ производителя в городе A
- Конечная точка: любой склад этого производителя (может быть в разных городах)
- Фильтр по видам транспорта (пользователь выбирает: машина/самолёт/корабль)

**Найти:**
- Топ-3 маршрута с минимальным **временем**
- Топ-3 маршрута с минимальной **ценой**
- Топ-3 маршрута с минимальным **расстоянием**

### Особенности графа

1. **Граф неориентированный** — дороги работают в обе стороны
2. **Граф взвешенный** — у каждого ребра 3 веса (время, цена, расстояние)
3. **Разные виды транспорта** — нужно учитывать фильтр пользователя
4. **Множественные критерии** — оптимизация по 3 метрикам одновременно

## Как запустить референсный проект

### Установка зависимостей
```bash
pip install Flask Flask-JWT-Extended Flask-SQLAlchemy Flask-CORS
```

### Первый запуск
1. Раскомментировать строку 39 в `main.py`: `СreateMap()`
2. Запустить: `python main.py`
3. Закомментировать обратно: `# СreateMap()`

### Обычный запуск
```bash
python main.py
```

Сервер запустится на `http://127.0.0.1:3001`
